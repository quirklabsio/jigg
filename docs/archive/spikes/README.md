# Archived Spike Documents

These are historical design research documents from the Jigg development process. Spikes are time-boxed research sessions that explore technical approaches before implementation.

## Status: Historical

**These docs are intentionally preserved but not kept current.** They document the decision-making process but may differ significantly from what was actually implemented.

## For Current Implementation

- **Keyboard accessibility:** Read `docs/accessibility.md` 
- **Piece tray rendering:** Read `docs/architecture.md` and `docs/drag-and-drop.md`

## Spike Files

- `spike-keyboard-focus.md` — Research for Stories 39-42 (keyboard navigation)
- `spike-tray-rendering.md` — Research for Stories 30-37 (piece tray architecture)

Each spike includes the research question, approaches considered, and final recommendation. The "Status: Complete" marking means a decision was reached, not that the spike content reflects shipped code.
